### Dynamic Component [https://www.gymcoding.co/9073c8a8-1e3f-41a7-b7b5-4384f7dd03cd/]
: 컴포넌트를 동적으로 변경하고 싶을 때 사용, 탭 인터페이스와 같이 동적으로 전환해야할 때 유용함
  (v-bind):is 속성 사용
  위 속성으로 전달할 수 있는 값
    1. 등록된 컴포넌트의 이름 string
    2. 실제 가져온 컴포넌트 객체 object
  
  * 이 때, 여러 컴포넌트간 전환 시 마운트가 매번 해제되며, 내장 컴포넌트 <KeepAlive>를 사용하여 비활성 컴포넌트들의 활성 상태를 유지할 수 있음
  * 일반 ref를 사용하는 것 보다 shallowRef를 사용하는게 퍼포먼스 적으로 유리함
    -> shallowRef
      : 반응형 객체임은 ref와 동일하지만 속성에 대해서 반응하지 않고 값 자체가 변경되었을 때만 반응함

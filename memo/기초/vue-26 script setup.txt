### scrtip setup [https://www.gymcoding.co/089bd27a-a030-4a5c-9096-ca2b76625455/]
: <script setup>은 SFC 내에서 CompositionAPI를 사용하기 위한 syntactic sugar(문법적 설탕)
  SFC와 CompositionAPI 사용 시 <script setup> 사용 하는 것을 권장, 일반 script 구문보다 많은 장점 제공
    - 상용구 감소
    - type script 사용한 props, emits 선언 가능
    - runtime 성능 향상

  * syntactic sugar(문법적 설탕) : 기능은 그대로인데 읽는 사람을 위해 직관적으로 코드를 읽기 쉽게 만든는 것

  컴포넌트를 처음 가져올 때 한 번만 실행되는 일반 <script>와 달리, <script setup>는 컴포넌트의 인스턴스가 생성될 때마다 내부 코드가 실행됨


## define Props()/Emits()
: <script setup> 사용 시 props, emits값을 가져오는 함수, <script setup> 내부에서만 사용할 수 있는 컴파일러 매크로

## defineExpose()
: 기본적으로 <script setup>사용하면 기본적으로 컴포넌트간 통신이 닫혀 있음, 컴포넌트 내부의 데이터나 메서드를 명시적으로 노출하기 위해 사용

## use Slots()/Attrs()
: <template> 내부에서 $slot/attrs로 직접 접근하여 사용 가능, <script>내부에서 사용 시 호출하는 함수


## 함께 사용
: <script setup>은 일반<script>와 함께 사용할 수 있음
  - <script setup>에서 표현할 수 없는 inheritAttrs옵션(상속여부 결정하는 속성), 플러그인을 통해 활성화된 옵션을 사용하고자 할 때
  - 한 번만 실행되어야 하는 로직이 있을 때


## Top-level await
: <script setup> 내 Top-level에서 await를 사용할 수 있음, 이러한 코드는 async setup()으로 컴파일 됨

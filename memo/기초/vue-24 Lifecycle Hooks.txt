### Lifecycle Hooks [https://www.gymcoding.co/64b3c1be-c017-44f2-b8ed-b473506e2981/]
: Lifecycle 단계에서 사용자가 프로그래밍 코드를 추가할 수 있는 단계별 기능, 즉 함수를 말함

* Lifecycle : 각각의 컴포넌트 인스턴스는 생성과 소멸될 때 사전에 정의된 몇 단계의 과정을 거치는 것


## Hooks
: compositionAPI에서는 optionsAPI와 다르게 beforeCreate, created 훅은 제공하지 않지만 setup함수 내에서 구현 가능


## Lifecycle 4 step
: Creation(생성) --> Mounting(장착) --> Updating(변경) --> Destruction(소멸)

  - Creation : 컴포넌트 초기화 단계, 가장 먼저 실행
                > 컴포넌트가 DOM에 추가되기 전, 접근 X
                > 서버렌더링에서 지원되는 단계
                > 클라이언트, 서버 렌더단에서 처리해야 할 일이 있을 경우 현재 단계에서 진행

                ex> beforeCreate(컴포넌트 인스턴스 초기화될 때, only optionsAPI), created(초기화 완료 후, only optionsAPI), setup(optionsAPI의 훅보다 먼저 호출됨)

  - Mouting : DOM에 컴포넌트를 삽입하는 단계
                > 서버렌더링에서 지원되지 않음
                > 초기 렌더링 직전에 DOM을 변경하고자 한다면 현재 단계 활용
                > Creation은 부모에서 자식 순으로 진행되지만, Mounting의 경우 자식에서 부모 순으로 진행됨

                ex> onBeforeMount(컴포넌트가 마운트 되기 직전, DOM 접근 불가능)/사용권장하지않음, onMounted(컴포넌트가 마운트 된 직후, DOM 접근 가능)

  - Updating : 컴포넌트에서 사용되는 반응형 데이터가 변경되거나 어떠한 이유로 재렌더링이 발생될 때 호출
                > 디버깅 혹은 프로파일링 등 컴포넌트 재렌더링 시점 파악하기 위해 사용 

                ex> onBeforeupdate(반응형 상태 변경으로 컴포넌트가 DOM트리를 업데이트 직전에 호출), 
                    onUpdated(반응 상태 변경으로 컴포넌트가 DOM 트리를 업데이트 후에 호출, 하위 -> 상위 컴포넌트 순으로 호출됨)

                    * onUpdate훅에서 컴포넌트 상태 변경 시 무한 루프 발생 주의

  - Destruction : 해제/소멸 단계

                ex> onBeforeUnmount(컴포넌트가 마운트 해제되기 지전), onUnmounted(컴포넌트가 마운트 해제된 후)

### 컴포넌트 기초 [https://www.gymcoding.co/06ac9fe6-bd22-4435-85ce-0c843cac3d61/]


* 컴포넌트 : UI를 재사용할 수 있도록 정의한 것


## 컴포넌트 사용방법 : 컴포넌트 정의 -> 컴포넌트 등록 -> 컴포넌트 사용


# 컴포넌트 정의
 - SFC(Sinfle File Component) / default
 - 문자열(string Component)

# 컴포넌트 등록
 - 전역 : Vue 인스턴스에 컴포넌트를 등록하여 어디에서나 사용할 수 있음
          편리하지만 사용하지 않는 컴포넌트가 계속해서 최종빌드에 포함 됨(애플리케이션 파일의 크기가 불필요하게 커짐)
          소스코드를 볼 때 종속성 관계를 파악하기 어려움

 - 지역 : 현재 등록된 컴포넌트 영역 안에서만 사용할 수 있음

# 컴포넌트 사용
: 등록한 컴포넌트는 <template> 안에서 원하는 만큼 사용할 수 있음
  컴포넌트는 사용할 때 마다 해당 컴포넌트의 새 인스턴스가 생성됨. 즉 사용할 때마다 setup()함수가 실행된다는 것을 의미함


* Pascal case 사용


## 스타일 가이드

  - SFC 컴포넌트 명명 규칙
    Pascal or kebab
    ex> MyComponent.vue or my-component.vue

  - 베이스 컴포넌트 이름
    prefix 'Base', 'App', 'V-'
  
  - 싱글 인스턴스 컴포넌트 이름
    prefix 'The'

### 조건부 렌더링 [https://www.gymcoding.co/b4a12357-9785-4aae-ade1-d05eca2c13df/]
: v-if(v-if/v-else/v-else-if) 디렉티브를 사용하여 조건부로 블록을 렌더링 할 때 사용
  조건에 해당하지 않는 스크립트는 렌더링조차 되지 않음

  만약 여러개의 html요소를 v-if 디렉티브로 연결하고 싶다면 <template>을 사용할 수 있음


## v-show
: 요소를 조건부로 표시하는 또 다른 옵션 디렉티브
  렌더링은 되었지만 style의 display 속성 값이 변경됨, v-if 디렉티브와의 차이점

* v-if vs v-show
: 일반적으로 v-if는 전환비용이 높은 반면, v-show는 초기 렌더링 비용이 높음
  
  - v-if는 조건에 따라 전환할 때 블록 내부의 컴포넌트들이 제거되고 다시 생성됨, 초기 렌더링 시 조건이 거짓이면 아무 작업도 수행하지 않음
  - v-show는 css기반 전환으로 초기 조건과 관계없이 항상 렌더링 됨

  자주 전환해야한다 > v-show
  런타임 시 조건이 변경되지 않는다 > v-if

  
  * v-if와 v-for 동시 사용 피하기
    > style guide 확인
